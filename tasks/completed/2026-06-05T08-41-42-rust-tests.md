### src-tauri/tests/integration.rs

Create this file. It uses the Tauri mock runtime to build a real app with state managed, then retrieves the state directly to drive end-to-end sequences.

```rust
use tauri::Manager;
use tauri::test::{mock_builder, mock_context, noop_assets};
use tempfile::TempDir;

fn build_test_app() -> (tauri::App<tauri::test::MockRuntime>, TempDir) {
    let dir = TempDir::new().unwrap();
    let db_path = dir.path().to_path_buf();
    let app = mock_builder()
        .setup(move |app| {
            let conn = healthy_lib::db::init(&db_path)?;
            app.manage(healthy_lib::db::AppState {
                db: std::sync::Mutex::new(conn),
            });
            Ok(())
        })
        .invoke_handler(tauri::generate_handler![
            healthy_lib::commands::entries::add_entry,
            healthy_lib::commands::entries::get_entries,
            healthy_lib::commands::entries::delete_entry,
            healthy_lib::commands::settings::get_setting,
            healthy_lib::commands::settings::set_setting,
            healthy_lib::commands::export::export_csv,
            healthy_lib::commands::observations::get_observations,
        ])
        .build(mock_context(noop_assets()))
        .expect("failed to build test app");
    (app, dir)
}
```

Note: `healthy_lib` is the crate name (from Cargo.toml `[lib] name = "healthy_lib"`). All `pub(crate)` functions in db and commands modules need to be `pub` for the integration test crate to access them — change all `pub(crate)` in the extracted functions to `pub`.

Four integration tests:

- `test_entry_lifecycle`: build app, get state, add 3 entries with distinct dates, call db_get_entries assert count==3, call db_delete_entry on the first, assert count==2
- `test_settings_roundtrip`: set api_url and api_model, get them back, assert values match; assert api_token returns None before being set
- `test_csv_export_content`: add 2 known entries, call build_csv, assert header present, assert 2 data rows present, assert specific field values in first row
- `test_app_state_is_managed`: build app, call `app.state::<healthy_lib::db::AppState>()`, lock the mutex, assert no error (confirms setup ran correctly)
