### src-tauri/src/db/migrations.rs
Two public SQL string constants:

`CREATE_ENTRIES`: creates table `entries` with columns:
- `id INTEGER PRIMARY KEY AUTOINCREMENT`
- `date TEXT NOT NULL UNIQUE` (ISO date, YYYY-MM-DD)
- `weight_kg REAL`
- `bp_systolic INTEGER`
- `bp_diastolic INTEGER`
- `energy_level INTEGER` (0–5, where 0 = fatigue, 5 = healthy)
- `notes TEXT`
- `created_at TEXT NOT NULL DEFAULT (datetime('now'))`

`CREATE_SETTINGS`: creates table `settings` with columns:
- `key TEXT PRIMARY KEY`
- `value TEXT NOT NULL`

### src-tauri/src/db/schema.rs
Two structs, both deriving `serde::Serialize`, `serde::Deserialize`, `Debug`, `Clone`:

`Entry`: fields matching entries columns (id: i64, date: String, weight_kg: Option<f64>, bp_systolic: Option<i64>, bp_diastolic: Option<i64>, energy_level: Option<i64>, notes: Option<String>, created_at: String). Add `#[serde(rename_all = "camelCase")]`.

`Setting`: key: String, value: String.

### src-tauri/src/db/mod.rs
- `pub mod migrations; pub mod schema;`
- `pub use schema::{Entry, Setting};`
- `pub struct AppState { pub db: std::sync::Mutex<rusqlite::Connection> }`
- `pub fn init(path: &std::path::Path) -> rusqlite::Result<rusqlite::Connection>`: opens (creates if absent) the DB at `path/healthy.db`, runs both migration SQL statements, returns the connection.
